# Bindraw — 独立画板 + 可选前置摄像头手势（小朋友版）

Bindraw 是 [yanxp.github.io](https://yanxp.github.io) 下的一个独立子项目：
**打开页面就能画画**，无需摄像头。想用手势作画时再勾选「启用手势」，才会
请求摄像头权限 + 加载手势识别模型。

- **默认模式（无摄像头）**：点格纹淡灰背景作为画纸，鼠标 / 触屏自由作画；
  工具栏里的「素描模板」提供 6 组动物线稿，小朋友可以在上面描色。
- **可选模式（前置摄像头手势）**：勾选「启用手势」后请求摄像头权限、加载
  MediaPipe Hands 模型。透明画布叠在摄像头画面之上，竖起食指跟随指尖作画。

核心思路：用 TensorFlow.js 的 `@tensorflow-models/hand-pose-detection`
（基于 MediaPipe Hands）实时检测手部 21 个关键点，做一个极简的二分类
（食指在画 / 抬笔）。默认不启动，避免在用户没有摄像头、没授权或只想用鼠标
画画时造成打扰。

## 界面

- **顶部工具栏**：画笔 / 橡皮擦、颜色、粗细、撤销 / 重做、清空、保存，
  **素描模板选择器**，以及三个可选开关：**启用手势（默认关）**、显示摄像头、
  显示骨架。
- **主画布**：
  - 无摄像头模式：淡灰格纹画纸背景，画笔线条清晰可见。
  - 手势模式：切换后摄像头视频铺满工作区并水平镜像；画布叠在上层，
    左上角显示手势徽章、左下角显示状态栏、右上角显示手势说明。
- **加载遮罩 + 就绪横幅**：只在用户勾选「启用手势」后才出现；失败
  （拒绝授权 / 无摄像头 / 模型下载失败）会自动复位回无摄像头模式，不会卡住。

## 手势控制（只需一个）

| 手势 | 动作 |
| --- | --- |
| ☝️ 竖起食指 | 下笔 — 跟随食指指尖连续作画 |
| 🖐 其它姿势 / 放下食指 | 抬笔（只显示光标，不画） |

其它操作（撤销、清空、保存）全部回到工具栏按钮，避免手势误触，适合给小
朋友使用。分类器偏袒「作画」：只要食指伸展、且中 / 无名 / 小指合计不超过
1 根伸展，就判为「下笔」；其它情况一律抬笔。

> 摄像头画面做了水平镜像以贴近用户直觉；识别坐标随后做水平翻转 +
> 指数移动平均（α = 0.45）再映射到画布，让指尖轨迹跟手且不抖。
> 手消失时 EMA 状态会重置，重新出现时不会在旧位置留下拖尾。

## 素描模板（动物线稿）

工具栏的「素描模板」下拉菜单提供 6 组简笔动物线稿，选中后以浅灰色画到
画布上，小朋友可以在上面自由着色：

- 🐱 小猫
- 🐰 小兔
- 🐟 小鱼
- 🦋 蝴蝶
- 🦕 恐龙
- 🐢 乌龟

> 所有线稿都是内联 SVG，直接打包在 `bindraw.js` 中，无第三方依赖。
> 选择新模板会提示清空当前画作；选择「无模板」会清空画布还原纯净底图。

## 模型加载性能

- 首屏会下载 **2–5 MB** 的 MediaPipe Hands 模型，一般 **2–10 秒**完成
  （视网络而定），第二次访问由浏览器缓存直接命中。
- 加载时中央有遮罩显示当前阶段：请求摄像头权限 → 下载模型 → 识别中。
- 模型就绪后：
  - 中央弹出绿色「✅ 准备就绪」横幅（2.8 秒后自动消失）。
  - 页面标题切换为 `✅ 就绪 — …`。
  - 左下状态栏显示「✅ 就绪 · 请把手伸到摄像头前」。
- 优先尝试 `mediapipe` runtime（加载快、鲁棒性好），失败则回退 `tfjs`
  lite / full。

## 功能

- **前置摄像头手势识别**：MediaPipe Hands（首选 `mediapipe` runtime，
  失败回退 `tfjs`）在覆盖层 canvas 上绘制骨架与指尖高亮。
- **手势驱动画板**：二态分类（食指作画 / 抬笔），带作画粘滞防断线。
- **动物素描模板**：6 组内置线稿（猫 / 兔 / 鱼 / 蝶 / 恐龙 / 龟）。
- **手动工具栏**（鼠标 / 触屏兼容，可与手势并用）：
  - 画笔 / 橡皮擦、颜色、粗细滑块
  - 撤销 / 重做（`Ctrl/Cmd+Z`、`Ctrl/Cmd+Shift+Z`）
  - 清空、导出 PNG（`Ctrl/Cmd+S`）
- **开关**：手势控制、显示摄像头、显示骨架。
- **响应式布局**：全屏自适应；跟随系统的深色模式。

## 使用方法

### 方式一：本地启动（推荐）

```bash
cd bindraw
python3 -m http.server 8000
# 浏览器访问 http://localhost:8000/
```

> 直接 `file://` 打开 `index.html` 部分浏览器会拒绝访问摄像头；请使用
> 本地 HTTP 服务或 HTTPS 环境。

### 方式二：GitHub Pages

仓库开启 GitHub Pages 后直接访问：

```
https://yanxp.github.io/bindraw/
```

GitHub Pages 默认提供 HTTPS，摄像头 API 可正常使用。

## 技术栈

- 原生 HTML / CSS / JavaScript（无前端框架）
- [TensorFlow.js](https://www.tensorflow.org/js) core / converter / webgl backend
- [@mediapipe/hands](https://www.npmjs.com/package/@mediapipe/hands)（mediapipe runtime 依赖）
- [@tensorflow-models/hand-pose-detection](https://github.com/tensorflow/tfjs-models/tree/master/hand-pose-detection)
- HTML5 Canvas 2D API + 内联 SVG 模板

## 实现要点

### 手势分类（camera.js）

极简二分类：

1. 每根手指做「伸展/收起」二分类：
   - 食指 / 中指 / 无名指 / 小指：`dist(tip, mcp) > dist(pip, mcp) * 1.35`。
   - 拇指：`dist(tip, mcp) > dist(ip, mcp) * 1.05`。
2. 判定：食指伸展 AND `(中 + 无名 + 小)` 合计 ≤ 1 → `point`，其它 → `none`。
3. 每 30 帧打印一次 `[t,i,m,r,p]` 与当前手势到 console，方便调试。

### 指尖到画布的坐标映射（bindraw.js）

1. camera.js 输出的 `fingertip.nx` 已做水平镜像（`1 - x/W`）匹配镜像显示。
2. 在画板侧用指数移动平均（α = 0.45）平滑 `nx, ny`。**手消失时重置 EMA
   状态**，重新出现时不会从旧位置拖线。
3. 根据当前手势二分处理：
   - `point`：`gesturePenDown()` → `gesturePenMove()`；抬笔时 `saveHistory()`。
   - `none`：抬笔，仅显示光标。
4. **作画粘滞**：进入 `point` 后，连续 5 帧内的偶发 `none` 误判仍当作
   `point` 处理，避免手稍微抖动就断线。

### 素描模板（bindraw.js）

- 6 组动物线稿都是内联 SVG（`viewBox 400×300`、浅灰 `#9ca3af` 描边）。
- 选中模板 → 通过 `Blob` + `Image` 加载 → `ctx.drawImage` 等比居中放到画布。
- 切换前若画布已有内容会弹 `confirm()` 提示清空。
- 模板画上去后作为新的「基态」压入历史，后续撤销也不会把模板擦掉。

## 注意事项

- **HTTPS / localhost**：浏览器仅在安全上下文下允许访问摄像头。
- **模型加载**：首屏会下载数 MB 的模型（约 2–10 秒）；就绪后会弹出显眼的
  「✅ 准备就绪」横幅。
- **隐私**：视频帧与识别全部在浏览器本地完成，不会上传到任何服务器。
- **识别鲁棒性**：极简二分类对单手竖直面向摄像头最稳；侧向或快速移动时
  可能出现短暂误判，但作画粘滞会吃掉偶发抖动。
- **诊断**：若识别不稳，可打开开发者工具查看 `[bindraw] fingers = ... gesture = ...`
  日志，确认当前手指状态是否被正确分类。
